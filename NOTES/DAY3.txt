## Login API

Endpoint:

```http
POST /login
```

Purpose:

Authenticate existing users.

---

## Login Flow

```text
Receive Email & Password
        ↓
Find User in MongoDB
        ↓
Check User Exists
        ↓
bcrypt.compare()
        ↓
Generate JWT Token
        ↓
Return Token
```

---

## User Lookup

```javascript
const user = await User.findOne({ email });
```

Purpose:

Find user using email.

---

## Password Verification

```javascript
bcrypt.compare(
    password,
    user.password
);
```

Purpose:

Compare entered password with hashed password stored in database.

---

## JWT Generation

```javascript
const token = jwt.sign(
    {
        userId: user._id
    },
    process.env.JWT_SECRET,
    {
        expiresIn: '1h'
    }
);
```

Purpose:

Generate authentication token after successful login.

## Interview Questions

### What is JWT?

JSON Web Token is a token-based authentication mechanism used to verify users.

---

### Why use JWT?

* Stateless authentication
* Secure
* Common REST API practice

---

### What is bcrypt?

A password hashing library used to securely store passwords.

---

### Difference between bcrypt.hash() and bcrypt.compare()?

```javascript
bcrypt.hash()
```

Creates hash.

```javascript
bcrypt.compare()
```

Verifies password.
